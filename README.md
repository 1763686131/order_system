# 📦 工业级订单管理看板系统 (局域网纯本地高可用版)

这是一个专门为工业和仓储作业环境设计的**高可用、低延迟、完全零外部依赖**的订单与业务状态流转看板管理系统。系统针对局域网断网环境进行了极致优化，前端无任何外部 CDN 依赖，具备严密的防呆/防误触机制、岗位权限隔离以及高精度业务双时间戳追踪系统。

---

## ✨ 核心系统特性

### 1. 🌐 100% 纯本地断网运行
* **零外网依赖**：移除了所有在线 CDN（如 Vue、Axios 等外部链接），前端完全基于原生 JavaScript (Vanilla JS) 与现代 CSS 开发，使用内置的 `fetch` API 与后端通信。
* **自适应静态托管**：后端 Flask 采用高级安全分发机制，完美适配 Linux/Docker 容器的大小写敏感路径，无缝支持 NAS 局域网部署。

### 2. 👥 严密的岗位权限隔离 (三级分工)
* **超级管理员 (super_admin)**：拥有最高大盘控制权。支持用户管理弹窗（可创建新用户、对操作员/管理员改密、物理注销账号）；同时拥有订单的**完整增、删、改、查（CRUD）**最高权限。
* **管理员 (admin)**：可查看全量大盘数据，允许快速发单与上传新订单，保留状态变更确认权限。
* **操作员 (operator)**：仅能查看当前时间视窗内的“未完成订单”。切换至历史已完成视图时，**所有业务动作按钮自动强行隔离隐藏**，变为纯只读状态，防止对已归档数据造成二次污染。

### 3. 🛡️ 工业级防误触/防呆设计
* **居中弹出可视化卡片**：点击“完成业务”或“设为未完成”时，系统会自动锁定背景重绘，并在屏幕正中央弹出一张与当前被操作卡片**内容完全一致的微缩版卡片**供核对。有效杜绝了因自动轮询卡片移位而导致的盲点误触。
* **可拉伸长文本域**：发布订单输入框采用可自由上下拉伸的文本域（`textarea`），完美适配各种超长货品参数、编号或地址，且**完美继承并识别键盘回车（换行符）**，排版清晰美观。

### 4. 📊 响应式响应看板与智能流转
* **卡片式网格大盘**：页面全屏撑满，两边干净利落毫无留白。订单采用卡片矩阵网格，每行最多自动铺 4 个（带窗口响应式自适应）。
* **状态颜色联动**：中固订单 pending 状态为标准白，绝缘订单 pending 状态为高亮粉色以示区分。只要点击确认完成，卡片一律**白/粉变绿**。
* **72小时动态时区窗口**：系统默认自动注入 `当前本地时间 - 72小时` 区间，自动过滤更早的历史订单，减轻一线人员视觉疲劳。

### 5. 🕒 高精度北京时间双时间戳
* **创建时间**：订单发布时，自动生成精确到小时和分钟的戳（如：`2026-07-11 13:05`）并置于卡片右上角。
* **完成时间**：点击完成业务时，时间戳印在卡片最左下角。若管理员执行了“回炉重造（变回未完成）”再完成，新分钟时间会**直接重写并完全覆盖**。

## 📂 项目文件目录结构

项目采用了非常清晰的模块化与多文件分离存储设计：

```text
order_system/
├── backend/                  # 后端服务端代码
│   ├── app.py                # Flask 高精静态分发路由及安全鉴权 API
│   ├── Dockerfile            # 本地 Docker 镜像轻量化构建文件
│   └── run.bat               # Windows 环境下一键秒启动批处理脚本
├── frontend/                 # 前端独立静态网页 (纯原生开发)
│   ├── css/
│   │   └── style.css         # 独立的看板全屏自适应布局样式表 (小写)
│   ├── js/
│   │   └── main.js           # 独立的断网业务处理、换行转义及防抖逻辑 (小写)
│   └── index.html            # 极精简的主骨架网页结构
└── data/                     # 数据持久化存储层 (多文件分离)
    ├── users_db.json         # 独立的用户账户表数据库
    └── orders_db.json        # 独立的业务订单看板数据库
