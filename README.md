# 📦 工业级订单管理看板系统 (局域网纯本地高可用版)

这是一个专门为工业和仓储作业环境设计的**高可用、低延迟、完全零外部依赖**的订单与业务状态流转看板管理系统。系统针对局域网断网环境进行了极致优化，前端无任何外部 CDN 依赖，具备严密的防呆/防误触机制、岗位权限隔离以及高精度业务双时间戳追踪系统。

---

## ✨ 核心系统特性

### 1. 🌐 100% 纯本地断网运行

* **零外网依赖**：移除了所有在线 CDN（如 Vue、Axios 等外部链接），前端完全基于原生 JavaScript (Vanilla JS) 与现代 CSS 开发，使用内置的 `fetch` API 与后端通信。
* **自适应静态托管**：后端 Flask 采用高级安全分发机制，完美适配 Linux/Docker 容器的大小写敏感路径，无缝支持 NAS 局域网部署。

### 2. 👥 严密的岗位权限隔离 (三级分工)

* **超级管理员 (super_admin)**：拥有最高大盘控制权。支持用户管理弹窗（可创建新用户、对操作员/管理员改密、物理注销账号）；同时拥有订单的**完整增、删、改、查（CRUD）**最高权限。
* **管理员 (admin)**：可查看全量大盘数据，允许快速发单与上传新订单，保留状态变更确认权限。
* **操作员 (operator)**：仅能查看当前时间视窗内的“未完成订单”。切换至历史已完成视图时，**所有业务动作按钮自动强行隔离隐藏**，变为纯只读状态，防止对已归档数据造成二次污染。

### 3. 🛡️ 工业级防误触/防呆设计

* **居中弹出可视化卡片**：点击“完成业务”或“设为未完成”时，系统会自动锁定背景重绘，并在屏幕正中央弹出一张与当前被操作卡片**内容完全一致的微缩版卡片**供核对。有效杜绝了因自动轮询卡片移位而导致的盲点误触。
* **可拉伸长文本域**：发布订单输入框采用可自由上下拉伸的文本域（`textarea`），完美适配各种超长货品参数、编号或地址，且**完美继承并识别键盘回车（换行符）**，排版清晰美观。

### 4. 📊 响应式响应看板与智能流转

* **卡片式网格大盘**：页面全屏撑满，两边干净利落毫无留白。订单采用卡片矩阵网格，每行最多自动铺 4 个（带窗口响应式自适应）。
* **状态颜色联动**：中固订单 pending 状态为标准白，绝缘订单 pending 状态为高亮粉色以示区分。只要点击确认完成，卡片一律**白/粉变绿**。
* **72小时动态时区窗口**：系统默认自动注入 `当前本地时间 - 72小时` 区间，自动过滤更早的历史订单，减轻一线人员视觉疲劳。

### 5. 🕒 高精度北京时间双时间戳

* **创建时间**：订单发布时，自动生成精确到小时和分钟的戳（如：`2026-07-11 13:05`）并置于卡片右上角。
* **完成时间**：点击完成业务时，时间戳印在卡片最左下角。若管理员执行了“回炉重造（变回未完成）”再完成，新分钟时间会**直接重写并完全覆盖**。

## 📂 项目文件目录结构

项目采用了非常清晰的模块化与多文件分离存储设计：

```text
order_system/
├── backend/                  # 后端服务端代码
│   ├── app.py                # Flask 高精静态分发路由及安全鉴权 API
│   ├── Dockerfile            # 本地 Docker 镜像轻量化构建文件
│   └── run.bat               # Windows 环境下一键秒启动批处理脚本
├── frontend/                 # 前端独立静态网页 (纯原生开发)
│   ├── css/
│   │   └── style.css         # 独立的看板全屏自适应布局样式表 (小写)
│   ├── js/
│   │   └── main.js           # 独立的断网业务处理、换行转义及防抖逻辑 (小写)
│   └── index.html            # 极精简的主骨架网页结构
└── data/                     # 数据持久化存储层 (多文件分离)
    ├── users_db.json         # 独立的用户账户表数据库
    └── orders_db.json        # 独立的业务订单看板数据库

```

### 部署与维护指南

💻 方案 A：Windows 本地电脑调试与运行
确保本地电脑已安装并启动 Docker Desktop。

进入 order_system/backend/ 目录，双击运行 run.bat。

批处理脚本会自动清理旧镜像并执行秒构建。启动成功后，系统会自动调用默认浏览器弹出前端控制面板：http://localhost:7899

💾 方案 B：NAS (群晖 / 威联通 / 绿联 / 飞牛OS) 正式环境部署
推荐使用 Docker Compose 方案进行图形化一键部署，不仅稳定，而且极其方便后期无缝修改维护：

将整个 order_system 项目文件夹上传到 NAS 的 docker 共享目录下。

打开 NAS 的 Docker 管理软件，选择新建 项目 (Project) 或 Compose 应用。

粘贴并应用以下配置（注意核对你的 NAS 实际绝对存储路径）：

```YAML
version: '3'

services:
  order-board:
    image: python:3.10-slim
    container_name: nas_order_board
    ports:
      - "7899:7899"
    volumes:
      # 请将冒号前的路径修改为你 NAS 上对应的绝对实体路径
      - /vol2/1000/Docker/order_system/backend:/app
      - /vol2/1000/Docker/order_system/data:/app/data
      - /vol2/1000/Docker/order_system/frontend:/app/frontend
    working_dir: /app
    # 🎯 强力矫正：注入中国北京本地东八区时区，确保发单与完成时戳精确一致
    environment:
      - TZ=Asia/Shanghai
    command: sh -c "pip install flask flask-cors -i [https://pypi.tuna.tsinghua.edu.cn/simple](https://pypi.tuna.tsinghua.edu.cn/simple) && python app.py"
    restart: always

```

### 🛠️ 后期开发维护与修改说明

得益于系统的双向卷挂载设计，容器内的运行实体就是你 NAS 上的真实文件，后期维护非常简单：

修改前端排版、调字号或改 JS 交互逻辑：

直接用文本工具修改 NAS 里的 frontend/css/style.css 或 frontend/js/main.js。

修改完后不需要重启 Docker 容器。直接去浏览器页面按 Ctrl + F5 强制刷新，全新的改动就会瞬间应用。

修改后端接口逻辑 (app.py)：

修改 NAS 里的 backend/app.py 并保存。

修改完后，在 NAS Docker 界面找到 nas_order_board 容器，点击 “重启 (Restart)” 按钮，1秒内即可载入新代码。

数据备份：

定时备份 data/ 文件夹下的两个 JSON 文件即可。即使升级或删除 Docker 容器，历史订单和账号密码也绝对安全。
